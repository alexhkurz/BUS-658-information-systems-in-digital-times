Week 7 Question: Sayde Pianko

If AI agents are going to make stipulated transactions instantly with other agents, how are we going to value the principles that people have such as being "Clean air focused" and how will we standardize human values in a way that makes this instantaneous complex agentic economy work?

 

This is a genuinely hard problem — it sits at the intersection of mechanism design, moral philosophy, and distributed systems. Let me work through it carefully.

The Core Tension

The problem is essentially this: human values are fuzzy, contextual, and contested — but agentic economies require precise, executable preference representations. You can't run a smart contract on "I care about clean air."

How Values Could Be Encoded

1. Preference Registries (Personal Value Profiles) Individuals could maintain a signed, versioned preference manifest — something like a structured file that says:

  • Carbon cost weight: 0.3 (normalized against price)
  • Local sourcing preference: strong
  • Labor standards floor: ISO 26000 compliant suppliers only

Agents acting on your behalf query this registry before executing transactions. The challenge is who curates the ontology of values — that's inherently political.

2. Outcome-Based Proxies (Value Oracles) Rather than encoding abstract values, you attach agents to measurement systems. A "clean air" value becomes a live feed from EPA AQI data, satellite NO₂ monitoring, or third-party environmental audits. The agent bids differently based on real-time environmental impact scores of counterparties.

This is more tractable but has a huge problem: whoever controls the oracle controls the value.

3. Stakeholder-Weighted DAOs Decentralized Autonomous Organizations could let affected communities vote on environmental impact weightings continuously. An agent transacting in a region would have to query the local DAO's current preference weights. This is democratic but extremely slow relative to millisecond agentic transactions — which creates a latency mismatch.

The Standardization Problem

This is where it gets philosophically serious. There are roughly three camps:

Government-led standardization Agencies like the EPA, SEC (for ESG), or a future "Value Standards Bureau" define canonical metrics. Pros: democratic legitimacy, enforceability. Cons: jurisdictional fragmentation, regulatory capture, values frozen at moment of legislation.

Market-emergent standardization Values get priced in through reputation systems and revealed preference — similar to how credit scores emerged. The market discovers what "clean air focused" is worth in aggregate. Pros: adaptive, no central authority. Cons: systematically undervalues things markets are bad at pricing (long-horizon, diffuse, non-excludable goods like air quality).

Protocol-layer standardization (the most interesting) An open standard — think IEEE for values — where anyone can propose a value module, communities ratify versions, and agents declare which version of a value ontology they're running. Like semantic versioning for ethics. This preserves pluralism: a "clean air v2.3" in California vs. "clean air v1.8" in Texas could coexist, and agents negotiate compatibility.

Making It Instantaneous

The speed problem is real. The likely architecture is two-layer:

Layer Speed Function
Slow layer Days/weeks Humans deliberate, vote, update value profiles and oracle weights
Fast layer Milliseconds Agents execute within pre-committed value constraints, like a constitution

This mirrors how constitutional law works: you don't re-litigate free speech every time a newspaper publishes. Agents operate within a pre-ratified value envelope, and edge cases get kicked to human review queues.

The Deepest Problem: Value Incommensurability

Even if you solve the technical layer, some values cannot be reduced to a common unit without doing violence to them. How many dollars of clean air is worth one unit of local job preservation? Different communities will answer this differently — and legitimately so.

The honest answer is that no fully automated system can resolve this. The best architecture probably looks like:

  • Agents handle routine transactions within well-mapped value territory
  • Contested tradeoffs trigger human escalation or community governance votes
  • Red lines (hard ethical constraints) are non-negotiable at the protocol level — not subject to price signals at all

What's Most Likely to Actually Happen

Realistically, the first iteration will be messy — large platforms (think AWS, Google, or a dominant agentic middleware) will define de facto value standards, just as they've done with content moderation. This will be contested, leading to regulatory intervention and eventually something like multi-stakeholder governance bodies — similar to how internet standards (IETF, W3C) evolved.

The clean air principle specifically is probably best handled not as a preference to be traded off, but as a floor constraint enforced at the infrastructure level — agents simply cannot execute transactions below a certain environmental threshold, full stop. That removes it from the market and puts it back in the domain of law and collective governance, which is arguably where it belongs.

The most important design principle: not everything that matters should be monetizable. A healthy agentic economy will need explicit "no-trade zones" — values held as inviolable rather than optimized.
